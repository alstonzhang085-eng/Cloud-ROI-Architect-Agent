🚪 API Layer
  • Client App - Mobile application for users to browse videos
  • API Gateway - Entry point handling authentication, rate limiting, and routing
  • Redis Cache - High-speed caching layer for frequently accessed feeds
⚙️ Service Layer
  • Feed Service - Orchestrates feed generation and serves requests
  • Candidate Service - Generates 2000 candidate videos using recall strategies
  • Ranking Service - Applies ML models to rank and select top 10 videos
🗄️ Data Layer
  • Feature Store - Centralized repository for real-time user and video features
  • Metadata DB - Stores video metadata, creator info, and content attributes
  • Video Index - Search index for fast video lookup and filtering
  • Object Storage - Stores actual video files and media content
📡 Messaging Layer
  • Kafka Queue - Streaming platform for real-time event processing
  • Stream Processing - Real-time analytics and feature updates from user interactions
  • User Events - Engagement signals: likes, comments, shares, watch time

🔄 Data Flow Architecture

User Request

Client sends feed request → API Gateway authenticates → Feed Service orchestrates pipeline

Candidate Generation

Candidate Service retrieves 2000 videos using recall strategies from Video Index

Feature Fetching

Real-time features fetched from Feature Store for ML ranking models

ML Ranking

Ranking Service applies deep learning models to select top 10 videos

Real-time Updates

User events stream through Kafka → Stream Processing updates Feature Store in seconds

Cached Response

Results cached in Redis for 30 seconds to reduce latency and load
